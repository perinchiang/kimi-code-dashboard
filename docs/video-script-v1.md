# Kimi Code Dashboard 视频脚本（v1）

时长：约 2 分 10 秒  
目标受众：已经在使用 Kimi Code CLI 的用户  
核心定位：解决 Kimi Code CLI 管理分散、状态不可视的痛点  

---

## 0:00 - 0:12｜开场：痛点先行

**画面**  
浏览器全屏展示 Dashboard 首页，从顶部 Token 趋势缓慢下拉到 mini cards 区域。

**你**  
Kimi Code CLI 功能越来越强，但用久了会发现一个问题：skills、MCP、定时任务、token 用量这些东西，都散落在不同的地方，想看个状态总得翻文件或者靠记忆。

**字幕**  
用 Kimi Code CLI 的你，有这感觉吗？

**注意**  
- 不要提 Cloud Code、Cursor、Claude、DeepSeek 等竞品  
- 不要解释技术架构或文件名  

---

## 0:12 - 0:30｜一句话定位

**画面**  
鼠标依次划过首页 mini cards：Skills、MCP、Hooks、定时任务、第三方模型、产物浏览器、工具调用、模型用量。

**你**  
所以我给自己做了这个 Kimi Code Dashboard。一个本地运行的 Web 面板，把你的 Skills、MCP、Memory、定时任务、模型配置、产物、用量，全聚到一个页面。

**字幕**  
本地运行 · 一站式管理

---

## 0:30 - 0:55｜核心：Skill 管理

**画面**  
点击 Skills mini card 进入 Skills 页面。展示列表、启用/禁用开关、搜索框、详情弹窗。

**你**  
我一开始做它，其实就是因为 skill 管理。装了哪些、哪些是开着的、每个是干嘛的、最近调了几次，以前得自己翻配置，现在一眼就能看到。

**画面**  
点击一个 skill 的开关，状态实时变化；再点开一个 skill 查看详情描述。

**字幕**  
Skill 启停，点一下

---

## 0:55 - 1:15｜用量可视化

**画面**  
回到首页，依次点击 Token 趋势的“日 / 周 / 月 / 年” tab，鼠标 hover 在趋势图上显示具体数字。再切到工具调用 / 模型用量页面快速展示。

**你**  
还有就是用量。每天、每周、每月花了多少 token，哪个工具调得多、哪个模型占比高，都有统计。终于不是黑盒了。

**字幕**  
Token 花在哪，看得见

---

## 1:15 - 1:40｜隐藏亮点：一键启动 + 手机终端

**画面**  
点击右上角“启动 Kimi Web”按钮，按钮变为“打开 Kimi Web”，再点击跳转。

**你**  
右上角这个按钮也很好用，一键启动本地 Kimi Web，外网访问也能自动处理信任域名。

**画面**  
切换到手机录屏：打开 SSH 客户端（如 Termius），输入 `kimi-dashboard 2`，回车。

**你**  
更方便的是，我在手机终端输入 `kimi-dashboard 2`，也能直接启动。人在外面、SSH 连到家里机器，打开手机浏览器就能用。

**字幕**  
手机终端一句话启动

---

## 1:40 - 2:00｜其他顺手功能

**画面**  
快速闪过以下页面：MCP 状态、Memory 甜甜圈图、Hooks 编辑、产物浏览器（点开一张图片）、第三方模型配置。

**你**  
还有 MCP 状态、Memory 记忆分布、Hooks 编辑、产物浏览器、第三方模型配置这些。产物浏览器里看到图片，还能一键上传到自己配的图床。

**字幕**  
MCP · Memory · Hooks · 产物 · 模型配置

---

## 2:00 - 2:10｜收尾

**画面**  
回到 Dashboard 首页，定格。

**你**  
开源的，安装一条命令。如果你是 Kimi Code CLI 用户，应该能省不少事。链接放评论区。

**字幕**  
开源 · 一键安装 · 链接在评论区

# 拍摄与剪辑建议

## 不要说的话
- ❌ “虽然大家都在推 Cloud Code，但我还是选择了 Kimi”
- ❌ “Kimi Web 前端好看”
- ❌ “我 web coding 了一个面板”
- ❌ 任何竞品名字（Cursor、Claude、DeepSeek 等）
- ❌ 技术文件名：wire.jsonl、mcp.json、config.toml

## 建议做的事
- ✅ 前 5 秒必须出现 Dashboard 全貌，让观众知道“这是个看板”
- ✅ 每个功能配一个具体操作画面，不要干念
- ✅ 手机终端启动那段放慢节奏，确保命令和结果清晰可见
- ✅ BGM 用轻快 lo-fi / tech，不要太激昂
- ✅ 结尾用“链接在评论区，试试”代替“点赞关注”

## 记忆点设计
- 手机终端 `kimi-dashboard 2` 一键启动（最可能传播的点）
- Skill 开关“点一下”启用/禁用（最直观的操作）
- Token 趋势“终于不是黑盒”（切中用量焦虑）
